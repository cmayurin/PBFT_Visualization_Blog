Open source PBFT visualization

Many open source PBFT implementations were checked in github. However, many implementations are either implemented in a language team is not familiar with or have many dependency/bug issues and hence, are difficult to work with. The PBFT implementation that was chosen for this visualization, (Source Code) is the basic version where two clients and four replicas are present and a single message is being sent. This simple PBFT protocol was developed over HTTP, using Python3 and asyncio/aiohttp. This implementation creates different instances of client and replicas on various network ports in the localhost.

Following code changes were made to visualize the algorithm and blockchain creation

  • Created a front end by building the graphics framework of the website and the corresponding interactive API functions
  • Functions created to display the real time data during a phase and after a phase at each replica when the algorithm is run in the backend
  • Changed/added the published information so the front end can capture it and use the data to enhance the visualization of the transaction flow
  • Debugged and fixed an issue with the current implementation with regards to the failure to add a block to the blockchain after a certain interval(ckpt_interval) of messages are received
  • Debugged and fixed an issue with the current implementation with regard to the failure to add a blocks to the blockchain for the last replica
  • Simulated Byzantine behaviour
  • Added new features to the front end to visualize blockchain
  • Establish connection between front and backend
